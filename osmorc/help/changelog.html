The history of Osmorc

1.2.2 | 01-September-2010

  • It is now possible to specify the version of the framework when creating it. Osmorc will use the supplied version when launching PAX Runner.
  • Maven support: Dependencies of type "bundle" are now correctly imported.
  • Resolving large projects is now time limited and should no longer freeze the IDE.
  • Bundlor-Support: Osmorc can now build modules using a bundlor template file.
  • Osmorc will no longer try to start or bundlify non-java dependencies (e.g. Flex libraries).
  • Settings dialogs have been divided so project settings and application settings will be located at proper positions.
  • Settings dialogs have received an internal rewrite for much improved performance.

1.2.1 | 07-July-2010

  • You can now set up a default start level which will be used for starting your dependencies.
  • The run configuration dialog now has some spinners for selecting the start level.
  • You can now specify program parameters for pax runner in the run configuration.
  • Osmorc will no longer create facets for Maven projects which are not of type "bundle".
  • The facet importer will no longer use bundlor template manifests (template.mf) as source, but rather work on the generated manifests.
  • Exploded bundles are now correctly started up.
  • The bundle compiler will now issue a more usable message when a manifest file is missing.
  • A few issues with the bnd wrapper have been resolved.
  • Various reported exceptions have been fixed.

1.2.0 | 01-April-2010

  • Lots of internals have been changed and refactored.
  • Launching is now done solely with the help of Pax Runner, which is way more mature and stable than Osmorcs internal launcher.
  • Starting a command console is now supported for all frameworks.

1.1.1 | 18-March-2010

  • This release is another bugfix and stability improvement round.
  • Several annyoing critical bugs have been fixed. A few dialogs had usability improvements.

1.1.0 | 08-november-2009

  • This release focused on bugfixes and usability improvements.
  • Several dialogs have been reworked, to match IDEAs dialog style
  • Some hard to debug issues have been made aware to the compiler so it should complain about them now.
  • Lots of smaller bugfixes.

0.14.0 | 02-august-2009

  • Merged Equinox Run config into OSGi Bundles run config.
  • QuickFix that registers a Bundle Activator in the manifest file or Osmorc facet configuration.
  • Osmorc facet detection now only requires that a manifest file contains a Bundle-SymbolicName.
  • Osmorc now makes sure that a manifest file is not readonly before trying to change it.
  • Bundle compiler has been rewritten and now supports bnd files much better.
  • Facet configuration dialog has received a rework to make it more intuitive to use.
  • Lots of smaller bugfixes.

0.13.0 | 02-may-2009

  • Maven Support: Imports/Updates Osmorc facet from Maven POM
  • Maven Support: Checks Maven dependencies to be OSGi compliant
  • Maven Support: Integration with SpringSource public repository to find replacements for non-osgi libraries.
  • Library Bundling: default bundling rule corrected so it works out of the box for most scenarios
  • Lots of internal refactoring and bugfixes.
  • For the Equinox run configuration separate config and working directories can de defined.
  • Added a new option that will force always force a rebuild of the bundle jar.

0.12.0 | 22-feb-2009

  • Added autocompletion for known manifest header names.
  • Equinox 3.4 source bundles are now used correctly.
  • Reworked additional JAR contents section in facet configuration. Source files and directories are now selected via a file chooser and a destination name that will most times be correct is inferred.
  • Splitted the facet editor into smaller parts.
  • Changes to files belonging to the additional JAR contents of a bundle now trigger a rebuild of the JAR.
  • Added specific Equinox, Bnd and Spring DM headers to the headers known by Osmorc.

0.11.0 | 31-dec-2008

  • Equinox run configuration: Additional JVM parameters can be specified.
  • Equinox run configuration: Additional Equinox parameters can be specified.
  • Equinox run configuration: Runtime directory can be specified.
  • Equinox run configuration: Products and Applications based on Eclipse 3.1 and 3.2 can now be run.
  • A regular expressionn can be specified that filters out files from the additional JAR contents.

0.10.0 | 15-dec-2008

  • OSGi bundles run configuration: bundles which are part of the framework can now be started.
  • OSGi bundles run configuration: libraries which are themselves OSGi bundles can now be started.
  • Multiple rules can now be applied when building a library bundle.
  • Bundle version from facet dialog is now correctly built into the manifest.
  • Library cache can now be deleted with a button in library building dialog.
  • Fragment bundles are now handled correctly when manifest files are edited manually.
  • The classpath used by the run configurations to start the OSGi container now only contains what is needed to start the container.
  • Added spell checking for manifest header names.

0.9.0 | 16-nov-2008

  • Added integration of Concierge OSGi
  • OSGi bundles run configuration: bundle start levels can be set.
  • OSGi bundles run configuration: debug mode can now be switched on for every framework type.
  • OSGi bundles run configuration: boot delegation and system packages can be specified for Felix and Knopflerfish frameworks.
  • Eclipse Equinox run configuration: Checks whether an Equinox framework instance is set for the project and shows helpful error messages if this is not the case..
  • Rules for the creation of bundles for libraries can now be specified.
  • Bug fixes.

0.8.0 | 25-oct-2008

  • Ported to IDEA 8.

0.7.0 | 20-oct-2008

  • Running of Bundles in Eclipse Equinox.
  • Running of Eclipse Equinox applications and products
  • It is now possible to specify additional files and folders that are copied into the generated bundle jars.
  • Osmorc now can handle modules with multiple content roots.
  • Improved bundle dependency resolver that is used when the manifest files are generated by Osmorc.
  • Some bug fixes.

0.6.0 | 04-aug-2008

  • Running of Bundles in Knopflerfish and Apache Felix
  • Bundle JARs are built automatically when the corresponding module is built
  • Manifest files can be generated automatically when they are needed. Bnd is then used to generate them
  • Errors can now be reported into Osmorc's Jira with IDEA's error reporting mechanism.
  • A module with all framework instance libraries is now created and maintained.
  • Diverse bug fixes.

0.5.0 | 21-apr-2008

  • Fixed creation and removal of framework instances and their libraries
  • Invalid imports from a framework instance library are now recognized
  • An error message is now shown if the chosen base folder is not a valid base folder for the chosen framework integratot
  • Imports of classes and packages in the same bundle are now not marked as invalid anymore.
  • Added more documentation.

0.4.1 | 30-mar-2008

  • The uploaded ZIP didn't contain an Osmorc directory. IDEA didn't succeed to install the plugin therefore.

0.4.0 | 30-mar-2008

  • Framework instances can now be defined and used. Currently Eclipse Equinox, Knopflerfish and Apache Felix are supported
  • Autodetection of Osmorc facet.
  • Added a bit of documentation which is available through IDEA's help system.
  • If the Osmorc facet is added to a module that doesn't contain a manifest file, a manifest file is created for it.
  • Osmorc now handles invalid manifest files a bit more gracefully..

0.3.0 | 21-jan-2008

  • Integrated Eclipse Equinox for resolving of bundle dependencies.

0.2.0 | 21-dec-2007

  • Bug fixed: Import-Package and Export-Package only worked with one-level packages.
  • Added parsing of Bundle-Version.
  • Added syntax highlighting for attribute and directive names.
  • Added settings page for colors and fonts of manifest files.

0.1.0 | 13-dec-2007

  • Initial release
  • Export-Package, Import-Package and Require-Bundle work, but version information isn't yet parsed.
